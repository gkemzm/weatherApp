## 프로젝트 실행 방법

1. code clone
2. npm install
3. npm run dev

## 구현한 기능에 대한 설명
### 1.위치및 날씨정보를 화면에 표시하기
메인 기능입니다. </br>
접속시 main 화면에 바로 나오게 했으며 </br>
요구 사항인 현재 기온, 당일의 최저기온, 최고 기온, 시간대별 기온을 표기했습니다</br>
ui는 깔끔한 방향성을 선호하기에 가볍게 만들었으며 openweathermap이 제공하는 날씨 icon을 사용했습니다.</br>

### 앱 첫 진입시 사용자의 현재 위치를 감지하여 그 위치의 날씨 정보를 보여줍니다.
"geocoder"를 사용하여 접속자의 현재 좌표를 가져와  "openweathermap"에서 제공하는 api를 통하여 정보를 특정하였습니다.</br>
초기 진입 한정으로 한정으로 현재 지역이 나옵니다.</br>

다만 geocoder의 특성상 초기검색이 영문으로 진행되는데 이는 아래의 방법으로 한글제공이 가능할것 같습니다.</br>
1. JSON파일을 수정후 매핑 (정적 매핑이기에 수고로움이 듬)</br>
2. next.js의 api router를 통해 프롬프트 작성및 llm사용을 통한 지역명 변환 + UX/UI향상 (server단에서의 작업이기에 클라영향도가 낮고 내가 원하는 방식으로 답변내용 전달가능)</br>
3. 외부 api 추가로 인한 번역및 치환 (외부 api추가로인해 번들이 커지며 외부 의존도가 올라가기에 선호하지않음)</br>

1or2번이 적합하긴 하지만 과제 특성상 스택제한과 JSON파일 수정이 불가능하여 두었습니다.

### 검색기능
JSON파일을 기반으로 인풋에 리스트가 나오게 하였으며, 시, 군, 구, 동 단위로 검색이 가능합니다.</br>
검색or엔터로 submit이 가능하며 유효하지 않은 장소는 toast로 "해당 장소의 정보가 제공되지 않습니다"를 명시합니다.</br>
검색범위는 대한민국으로 한정지었습니다.</br>

### 즐겨찾기
해당 부분은 가볍게 storage를 사용해서 개발했습니다. </br>
만약 보안요소가 추가됐다면 쿠키를 로그인이 있었다면 db를 활용했을것 같습니다.</br>

최대6개까지 저장가능하며 alise 컬럼을 따로 생성하여 별칭관리를 합니다.</br>
추가, 삭제가 가능하고 주로 js 내장함수인 addEventListener와 react hook을 사용해 customhook을 만들어 사용했습니다. </br>

상세보기를 누를경우 상세페이지로 이동합니다.


## 기술적 의사결정 및 이유
### React18
React는 18버전을 사용하였습니다. </br>
주로 Next.js 14~15버전을 많이 다뤄왔기에 해당 버전에 대응되어 익숙하며, SSR작업이 필요할시 SSR작업에 편의성을 주는 React 18버전을 채용했습니다.

### react-router-dom
상세페이지가 존재하기에 확장가능성을 고려하여 MPA방식에서 라우팅을 편하게 관리할 수 있어 채택했습니다.  

### 위치 및 날씨 정보  "openweathermap", "geocoder"
해당 기능은 "openweathermap", "geocoder" 두가지 외부 API를 사용했습니다.</br>
"openweathermap"는 좌표를 기반으로한 기상정보를 받아오기 위함이고,</br>
"geocoder"지역 검색을 통하여 좌표를 받아오기 위함입니다. 

### tailwindcss 3.x
현재 tailwindcss의 최신 버전인 4버전은 일부 코드편집기에서 워닝을 일으키기에 가장 대중적인 3버전을 사용했습니다.

### react-hot-toast 
위치 조회시 ui를 표현하기 위해 사용했으며, react 시장에서 가장 사용량이 높은 라이브러리를 채택했습니다. 


채택이유는</br>

1. 구현 목표가 한국주소에 한정되어있지만, 실제 서비스일 경우 해외까지 확장성을 고려하여 기상청이 아닌  "openweathermap", "geocoder" 를채택했습니다.</br>
2. 가장 대중적이면서 부분적으로 무료로 사용할 수 있는 api를 채택하였습니다.</br>
